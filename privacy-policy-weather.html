Privacy Policy for Local Weather

Effective date: 2025-07-02

Local Weather ("the App") is a privacy-first weather application for Android and iOS that provides detailed forecasts while maintaining complete data sovereignty. This Privacy Policy explains how the App handles information. In summary:

  • Weather data is fetched from Open-Meteo services only when you request it.
  • All weather data and preferences are stored locally on your device.
  • You have complete control over your location data and weather history.

1. Information We Collect

1.1 Weather Data

The App fetches weather information from Open-Meteo services when you request forecasts:

  • Weather API: api.open-meteo.com - for current conditions and 72-hour forecasts
  • Geocoding API: geocoding-api.open-meteo.com - to convert location names to coordinates
  • Weather data includes temperature, humidity, precipitation, wind, and sunrise/sunset times
  • All weather data is cached locally for up to 7 days for offline access

1.2 Location Information

Location data is handled with complete user control:

  • You manually enter ZIP codes, city names, or coordinates - no automatic location detection
  • Location preferences and history are stored locally on your device
  • Multiple saved locations are supported for quick switching
  • Location data is only sent to Open-Meteo for weather lookups when you request them

1.3 App Settings and Preferences

The App stores the following data locally on your device only:

  • Theme preferences and visual settings
  • Saved locations and location history
  • Weather data cache and storage preferences
  • App configuration settings

1.4 Data We Do NOT Collect

  • No analytics, advertising, or crash-reporting data
  • No device identifiers or personal information
  • No background location tracking or GPS access
  • No account creation or user authentication
  • No usage statistics or behavioral tracking

2. Data Storage & Processing

  • All weather data and preferences are stored locally using SQLite database
  • Weather data is automatically cached for up to 7 days for offline functionality
  • On Android, data resides in app-specific directory (/Android/data/com.localweather.localweather/)
  • No cloud backup, synchronization, or remote servers for weather data storage
  • Data is only accessible by the App and not shared with other applications

3. Network Access & Third-Party Services

3.1 Open-Meteo Weather Services

The App connects to Open-Meteo services for weather data:

  • Purpose: Fetch current conditions and 72-hour forecasts
  • Data Sent: Geographic coordinates or location identifiers you provide
  • Data Received: Weather forecasts, current conditions, sunrise/sunset times
  • Privacy: Open-Meteo is a privacy-focused service with no user tracking or data collection
  • Frequency: Requests are made only when you explicitly refresh weather data

3.2 Intelligent Caching Strategy

The App implements smart caching to minimize network requests:

  • 1-hour cache validity for current conditions
  • Automatic fallback to cached data when network is unavailable
  • 7-day data retention for historical context and offline functionality
  • No unnecessary network requests - data is fetched only when needed

3.3 No Other Third-Party Integrations

The App contains no advertisements, analytics SDKs, or other third-party services.

5. Weather Data Privacy

  • User-Initiated: Weather requests are made only when you manually refresh or open the app
  • Location Control: You manually enter locations; no automatic location detection or GPS access
  • Data Retention: Weather data is cached locally for 7 days, then automatically expired
  • Offline Operation: App works entirely offline using cached weather data
  • No Tracking: Open-Meteo services do not track users or collect personal information

6. Offline Functionality

The App is designed for robust offline operation:

  • Cached weather data remains accessible for up to 7 days without internet
  • All app settings and location preferences work offline
  • Theme switching and app customization function without network access
  • Historical weather data browsing works entirely offline

7. Children's Privacy

The App is suitable for all ages including children under 13. Because minimal data is collected and no personal information is transmitted, it complies with COPPA requirements.

8. Security

We implement security best practices:

  • Local data is protected by Android/iOS app sandboxing
  • Network requests use HTTPS encryption
  • No sensitive data is logged or transmitted
  • Your device security (screen lock, encryption) protects your weather preferences

9. Your Choices & Control

  • Location Management: Add, edit, or remove saved locations at any time
  • Cache Control: Weather data automatically expires after 7 days
  • Network Control: Use the app entirely offline with cached data
  • Theme Preferences: Customize visual appearance without network access
  • Data Removal: Uninstalling the App permanently removes all local data

10. International Data Transfers

Weather data requests to Open-Meteo services may involve international data transfers as these services operate globally. Only location coordinates and weather data are involved in these transfers - no personal information.

11. Changes to This Policy

We may update this Privacy Policy to reflect App changes or legal requirements. Updated policies will be posted in-app and on our website with a new "Effective date."
